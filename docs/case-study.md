# ThreatLens-Kali Case Study: SSH Brute Force Investigation

## Overview

This case study demonstrates ThreatLens-Kali's capabilities in detecting, analyzing, and reporting a simulated SSH brute force attack on a Kali Linux system. The investigation follows a realistic incident response workflow.

**Environment:** Kali Linux 2025.2 workstation  
**Trigger:** Unusual authentication failures in system logs

## Detection Process

### Step 1: Log Ingestion
The `parsers.py` module loaded the `auth.log` file from the `sample_logs/` directory, parsing 20 authentication events.

### Step 2: IOC Extraction
The `ioc_extractor.py` module identified the following Indicators of Compromise (IOCs) from the log data:
* **IP Addresses:** 192.168.1.100, 172.16.0.10, 192.168.1.200
* **Source Ports:** 44287-44293, 44310-44312, 44320-44322

## Analysis Summary

The threat actor demonstrated a multi-stage attack pattern:
1. **Reconnaissance Phase:** Multiple failed SSH login attempts targeting high-value accounts (root, admin)
2. **Execution Phase:** Potential reverse shell establishment via netcat (`nc -e /bin/bash`)
3. **Privilege Escalation:** Unauthorized sudo access attempts from non-sudoers user
4. **Persistence:** Attempted access to system configuration files

## Risk Assessment

| Factor | Assessment |
|--------|------------|

| Likelihood | MEDIUM - Attack was detected before full compromise |
| Overall Risk | **HIGH** |

## Recommendations

### Immediate Actions
* Block malicious source IP addresses (192.168.1.100, 172.16.0.10)
* Reset compromised account credentials
* Review and revoke unauthorized sudo privileges

### Preventive Measures
* Implement SSH key-based authentication
* Configure fail2ban for automatic IP blocking
* Enable multi-factor authentication (MFA) for SSH access
* Monitor for unusual process execution patterns

### Monitoring
* Deploy continuous log monitoring using ThreatLens-Kali
* Set up automated alerting for critical severity events
* Conduct regular reviews of YAML detection rules

## Conclusion

ThreatLens-Kali successfully detected and analyzed a multi-stage attack targeting the Kali Linux system. The modular architecture enabled comprehensive threat detection through log parsing, IOC extraction, rule-based detection, MITRE ATT&CK mapping, and automated report generation (both Markdown and HTML formats).
